# ТЗ: AI Книга — Персонализированные детские сказки

## 1. Концепция

Веб-сервис, который создаёт персонализированные сборники детских сказок. Родитель загружает 2-3 фотографии ребёнка, вводит имя/пол/возраст, выбирает черты характера для развития — и получает PDF-книгу, где главный герой выглядит как их ребёнок.

---

## 2. Пользовательский флоу

```
1. Лендинг → кнопка "Создать книгу"
2. Загрузка 2-3 фото ребёнка (лицо, разные ракурсы)
3. Ввод данных: имя, пол, возраст
4. Выбор черт характера (дружба, храбрость, доброта, честность и т.д.)
   → Выбирает 3-5 черт из списка
   → На каждую черту — 3-5 сказок из пула 20 историй
5. Превью: обложка + 2-3 страницы бесплатно
6. Оплата (ЮKassa)
7. Генерация книги (1-5 минут)
8. Скачивание PDF / отправка на email
9. Кнопка "Заказать печатную версию"
```

## 3. Контент

### 3.1 Истории
- **Количество**: 20 историй (на старте)
- **Генерация**: AI (Claude/GPT) + ручная редактура и проверка педагогом
- **Категории черт характера**:
  - Дружба
  - Храбрость
  - Доброта
  - Честность
  - Трудолюбие
  - Щедрость
  - Уважение к старшим
  - Любознательность
  - Ответственность
  - Терпение
- **Персонализация текста**: имя ребёнка + пол (он/она, мальчик/девочка) + возраст (сложность текста)

### 3.2 Иллюстрации
- **Стиль**: зависит от конкретной книги (акварель, мультяшный, сказочный и т.д.)
- **Количество**: 5-10 иллюстраций на историю
- **Технология**: AI-генерация с лицом ребёнка (см. раздел 6)
- **Итого на книгу**: ~50-100 страниц (текст + иллюстрации)

### 3.3 Обложка
- Персонализированная: имя ребёнка + его изображение в стиле книги

## 6. Технологии генерации иллюстраций

### Подход: микс Pro + Flash (проверено тестами)

| Задача | Модель | Цена за шт | Время |
|--------|--------|-----------|-------|
| Иллюстрации с лицом ребёнка | **Nano Banana Pro** (gemini-3-pro-image-preview) | ~$0.24 (~22₽) | ~25 сек |
| Фоны, пейзажи, декор без лица | **Nano Banana Flash** (gemini-2.5-flash-image) | ~$0.039 (~3.6₽) | ~12 сек |
| Запасной вариант | Flux + PuLID (Replicate) | — | — |

**Результаты тестов (18.02.2026):**
- Pro значительно лучше сохраняет черты лица ребёнка — лицо узнаваемо
- Flash делает "обобщённого" ребёнка похожего типажа, но не узнаваемого
- Оба дают качественную акварельную стилизацию
- Вывод: Pro обязателен для сцен с лицом, Flash достаточен для фонов

### Структура книги:
- **5 историй** (по 1 на выбранную черту характера)
- **7 иллюстраций на историю** (5 с лицом + 2 фоновых)
- **1 обложка** (с лицом, Pro)
- **Итого: 36 иллюстраций** (26 Pro + 10 Flash)

### Процесс генерации одной книги:
```
1. Пользователь загружает 2-3 фото → сохраняем в Supabase Storage
2. Для каждой иллюстрации в выбранных историях:
   a. Берём заготовленный промпт (сцена + стиль)
   b. Если сцена с лицом → Gemini Pro API + референсные фото
   c. Если фон/декор → Gemini Flash API (без фото)
   d. Пост-обработка (ресайз, цветокоррекция)
3. Собираем PDF (текст + иллюстрации + обложка)
4. Отправляем пользователю
```

### Характеристики (по результатам тестов):
- **Время генерации 1 иллюстрации**: Pro ~25 сек, Flash ~12 сек
- **Стоимость AI на книгу**: 26 Pro × 22₽ + 10 Flash × 3.6₽ = **~610₽**
- **Время генерации книги**: ~3-5 минут (с параллельной генерацией)

### Важно: Geo-ограничения Gemini Image API
- API заблокирован в РФ и части ЕС
- Запросы проксируются через US-сервер (raw HTTP, не SDK)
- Обязательно IPv4 (`NODE_OPTIONS="--dns-result-order=ipv4first"`)

## 8. Структура базы данных

```sql
-- Пользователи (расширение Supabase Auth)
profiles (
  id UUID PRIMARY KEY REFERENCES auth.users,
  email TEXT,
  name TEXT,
  referral_code TEXT UNIQUE,
  referred_by UUID REFERENCES profiles(id),
  balance INTEGER DEFAULT 0,  -- скидочный баланс в рублях
  created_at TIMESTAMPTZ
)

-- Дети
children (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),  -- NULL если без аккаунта
  session_id TEXT,                         -- для гостей
  name TEXT NOT NULL,
  gender TEXT NOT NULL,  -- male/female
  age INTEGER NOT NULL,
  photos TEXT[],  -- URLs фото в Storage
  face_embedding JSONB,  -- данные лица для генерации
  created_at TIMESTAMPTZ
)

-- Истории-шаблоны
stories (
  id UUID PRIMARY KEY,
  title TEXT NOT NULL,
  trait TEXT NOT NULL,  -- черта характера
  content_template JSONB,  -- текст с плейсхолдерами {name}, {he_she} и т.д.
  illustration_prompts JSONB,  -- промпты для каждой иллюстрации
  style TEXT,  -- стиль иллюстраций
  page_count INTEGER,
  sort_order INTEGER,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ
)

-- Заказы
orders (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),  -- NULL если гость
  child_id UUID REFERENCES children(id),
  email TEXT NOT NULL,
  selected_traits TEXT[],
  selected_stories UUID[],
  status TEXT DEFAULT 'pending',
  -- pending → paid → generating → ready → downloaded
  payment_id TEXT,  -- ID платежа ЮKassa
  amount INTEGER,  -- сумма в копейках
  discount INTEGER DEFAULT 0,
  referral_code_used TEXT,
  pdf_url TEXT,
  print_ordered BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ
)

-- Печатные заказы
print_orders (
  id UUID PRIMARY KEY,
  order_id UUID REFERENCES orders(id),
  address JSONB,
  status TEXT DEFAULT 'pending',
  -- pending → in_production → shipped → delivered
  tracking_number TEXT,
  created_at TIMESTAMPTZ
)

-- Реферальные начисления
referral_transactions (
  id UUID PRIMARY KEY,
  referrer_id UUID REFERENCES profiles(id),
  referred_id UUID REFERENCES profiles(id),
  order_id UUID REFERENCES orders(id),
  amount INTEGER,  -- сумма начисления
  created_at TIMESTAMPTZ
)
```

## 10. Печать — план для MVP

**Этап 1 (MVP):** Полуавтоматический процесс
- Пользователь нажимает "Заказать печать" → вводит адрес доставки → оплата
- Заявка приходит в админку + на email
- Оператор вручную отправляет PDF в типографию
- Типографии-кандидаты в Москве:
  - **Printio.ru** — есть API, печать по одной штуке
  - **T-Print** — фотокниги, хорошее качество
  - **Принтбук (printbook.ru)** — специализация на фотокнигах, от 1 экз.
  - **Фотокнига.ру** — разные форматы, от 1 штуки
  - **Netprint** — широкая сеть, пункты выдачи

**Этап 2:** Автоматическая интеграция с типографией через API
